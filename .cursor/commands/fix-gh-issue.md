# GitHub Issue 修正コマンド

## 概要
GitHub issueの内容を取得し、解決のためのコーディング計画を提案します。

## 使用方法
issue番号を引数として渡してください（例: `42`）

## 手順

### 1. Issueの取得
以下のコマンドでissueの詳細を取得してください：

```bash
gh issue view $ARGUMENTS --json title,body,labels,assignees,state,comments
```

### 2. Issue分析
取得したissueから以下を分析してください：
- **問題の概要**: 何が問題なのか
- **再現手順**: バグの場合、どのように再現できるか
- **期待される動作**: 正しい動作はどうあるべきか
- **関連ファイル**: 影響を受けそうなファイルやモジュール

### 3. コーディング計画の提案
以下の形式でコーディング計画を提案してください：

#### 影響範囲の調査
- 関連するコードベースの調査
- 依存関係の確認
- 既存テストの確認

#### 実装計画
1. **修正対象ファイル**: 変更が必要なファイルをリストアップ
2. **具体的な変更内容**: 各ファイルで行う修正の説明
3. **新規作成が必要なファイル**: 必要に応じて
4. **テスト計画**: 追加・修正すべきテストケース

#### リスク評価
- 破壊的変更の可能性
- 副作用の可能性
- パフォーマンスへの影響

### 4. ブランチ作成
計画が承認されたら、以下でブランチを作成：

```bash
git checkout -b fix/issue-$ARGUMENTS
```

## 出力形式
```markdown
## Issue #[番号]: [タイトル]

### 問題の要約
[issueの内容を簡潔にまとめる]

### 影響を受けるコード
- `path/to/file1.ts`
- `path/to/file2.ts`

### 実装計画
1. [ステップ1の説明]
2. [ステップ2の説明]
3. [ステップ3の説明]

### テスト計画
- [ ] [テストケース1]
- [ ] [テストケース2]

### 開発者への確認事項
- [ ] [確認事項1]
- [ ] [確認事項2]
- [ ] [確認事項3]
```
